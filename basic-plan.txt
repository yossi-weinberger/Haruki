## 1) מטרת המטלה

לבנות אפליקציית Web בשם **Haruki**: “Personal Library / Wish List”
השראה לשם:

> “If you only read the books that everyone else is reading, you can only think what everyone else is thinking.” — Haruki Murakami (מופיע כציטוט שמיוחס ל־Norwegian Wood). ([Farnam Street][1])

יכולות חובה:

* חיפוש ספרים לפי כותרת/מחבר דרך Google Books API ([Google for Developers][2])
* הצגת תוצאות ככרטיסים (עטיפה, כותרת, מחברים)
* הוספה ל־Wish List, ומניעת הוספה כפולה (כפתור disabled/hidden אם כבר קיים)
* מסך Wish List עם הסרה
* שמירה מקומית שנשארת אחרי רענון (localStorage)
* ניווט בין Search ↔ Wish List

בונוסים:

* Debounce לחיפוש
* Loading / Error states + spinner/skeleton
* Book detail modal (תיאור/תקציר)
* מצב בהיר/כהה
* עברית/אנגלית + RTL

## 3) ארכיטקטורה מומלצת

### עקרון על

להפריד “דאטה עסקי” (Wish List) מהעדפות UI (Theme/Language), ולהשאיר Search כמצב מקומי.

### Stores (Zustand)

1. **wishlistStore (Business State)**

* state: `items[]`
* actions: `add(book)`, `remove(id)`
* selector/derived: `isInWishList(id)` או `idsSet` כדי להשבית Add מהר ובפשטות
* persist: localStorage (Zustand persist)

2. **settingsStore (UI Preferences)**

* `theme: light | dark | system` + persist
* `language: en | he` + persist

### Search state (לוקאלי בעמוד)

* `query`, `results`, `loading`, `error`
* Debounce לפני קריאה ל־API
* ביטול/התעלמות מבקשות ישנות (כדי שלא תוצאה ישנה “תנצח”)

### שכבת API נקייה

* `googleBooksClient` שמבצע fetch ל־Volumes endpoints ([Google for Developers][2])
* Adapter שממיר את התגובה למודל פנימי אחיד

### מודל נתונים פנימי (Book)

שמור רק מה שצריך ל־UI ול־Wish List:

* `id` (volumeId)
* `title`
* `authors[]`
* `thumbnail?`
* `description?`

## 11) פרומפט מוכן להדבקה לקלוד קוד (ללא קוד)

```text
Project name: Haruki
Build a React + TypeScript web app: search books via Google Books API and maintain a persistent Wish List.
Use: Zustand (with persist to localStorage), Tailwind + shadcn/ui, Framer Motion.
Quality: ESLint + Prettier from day 1. Add Vitest + React Testing Library only if time remains.

Core routes:
- /search (or /)
- /wishlist
- /about (a polished in-app README)

Brand & colors:
Light palette: {"bg":"#FFFBF5","fg":"#1C1917","muted":"#F5EFE6","border":"#E7DDCF","primary":"#F59E0B","accent":"#FBBF24","secondary":"#14532D"}
Dark palette:  {"bg":"#0F0D0A","fg":"#FAF7F2","muted":"#1A1713","border":"#2A241D","primary":"#FBBF24","accent":"#FCD34D","secondary":"#86EFAC"}

Architecture:
- wishlistStore (business state): items[], add/remove, selector isInWishList(id) or idsSet, persist localStorage.
- settingsStore (UI prefs): theme light/dark/system + language en/he, persist localStorage.
- Search state is local to SearchPage (query/results/loading/error) with debounced input.
- API layer: googleBooksClient + adapter to normalize response into Book:
  { id, title, authors[], thumbnail?, description? }.

About page requirements:
- Clean, short, modern. No emojis. No purple. No long marketing text.
- Include a short quote attribution line:
  “If you only read the books that everyone else is reading, you can only think what everyone else is thinking.” — Haruki Murakami
- Sections: Overview, Features (4–5 bullets), Architecture (4 bullets), Privacy (1 line), Credits (Google Books).
- Include subtle animations and respect prefers-reduced-motion.

Workflow:
- Create PRD.md, ARCH.md, TASKS.md first (and optionally AGENTS.md rules).
- Implement tasks one-by-one using Plan → Patch → Verify → Commit.
- For each task: propose plan + files, implement only that task, run lint/format, provide 2–3 manual test steps, then propose a concise commit message.
```
